Simple Postgres Database Setup

25 November 2015

This is a short and sweet post on how to set up your Postgres database in your terminal.

While there are many great resources out there to help you out once you've set up your database, the information seems to be scattered and for a newbie there are things that I didn't know about setting up a database. I hope that with this guide it will clear up some of the same confusions I've had and make setting up a database as painless as possible!

Before you start

*Make sure Postgres, pg, and sql (optional) are installed on your computer

Detailed installation guide for Postgres
Guide to install pg with npm
Guide to install sql with npm - You need this if you want to build your SQL statements with javascript.

**Double check that there is no Postgres database server running in the background if you already have Postgres installed.

How to shut down Postgres server

Now for the real fun

(The following steps are inside your terminal.)

1. Set Up (Create new folder):

This is a very important step. Your new database needs to be set up in an empty folder. Otherwise, other files will mess with the new database.

Locate your project folder inside the terminal.

Type the command: 

$ mkdir "foldername"

The $ sign in the commands is an indicator of a new line in the terminal.

Omit the double quotes " " when typing in your own folder name here and in following examples. The quotes are just to indicate the word quoted is a placeholder word.

mkdir creates an empty folder in the project's root directory.

Optionally, you can create a folder in your project's root directory in Finder.

2. Initialize Database:

Go to your project folder's root directory (this will be one level up from the database folder you have just created.)

Type the command: 

$ initdb "foldername"

3. Starting Database Server:

Type the command: 

$ postges -D "foldername"

4. Create Database Owner/User:

In a separate terminal window.

Type the command: 

$ createuser "name"

(The standard username is root, but it can be any name you want. Just remember what name you set your user to.)

5. Create the Database:

In the same terminal window as step 4

Type the command: 

$ createdb "databasename" -O "username"

*Alphabet 'O' as in Owner. Not the number zero.

(The -O "username" sets up the database to allow the owner with that username to configure and manage the database. This is good practice for working with a group. Just share the username with the rest of your group so they can have access to the database too.)

6. Get Inside Your Database:

In another terminal window

Type the command: 

$ psql "databasename"

(This command will take you inside your database environment where you can perform SQL tasks like insert a table, populate your table and query your data with SQL commands.)

There you have it. Six simple steps to setup your Postgres database and now off you go to have fun in SQL land!
